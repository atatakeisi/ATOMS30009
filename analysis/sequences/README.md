# sequences/ — Teaching シーケンス置き場（参照ライブラリ）

Teaching ファーム（`src/teach/`）の **⬇ エクスポート(JSON)** で吸い出した
キーフレームシーケンスをここに集める。さまざまな動きの**参照ロジック**として蓄積し、
Python 第2層（力学検証・最適化）や IMU 連携解析の入力にする。

## ファイル形式（自己完結JSON）

エクスポートJSONは単体で `raw → 角度 → 脚先(x,z)` まで再現できるよう較正情報を同梱する。

```json
{
  "name": "current",
  "defMs": 600,
  "calib": { "offset": [-35,0,13,-7,53,-44,15,34],
             "L1": 50.0, "L2": 65.0, "sign": "RL/FR:+ , RR/FL:-" },
  "frames": [
    { "t": 600, "raw": [511,511,524,504,564,467,526,545] }
  ]
}
```

- `raw[8]`: ID0..7 の生サーボ位置（0..1023）。較正非依存でそのまま再生可能。
- `t`: そのコマへ前コマ（先頭コマは現在位置）から移行する時間 [ms]。
- `calib`: raw→角度変換に必要。ID→脚/関節の対応は [docs/DESIGN.md](../../docs/DESIGN.md) 参照。

## 命名の目安

iPad ダウンロードは既定で `robot0009_seq.json`。Macへ移したら動きが分かる名前に変える:
`walk.json` / `sit.json` / `wave.json` など。

## 注意（形と力学の分離）

ここに入るのは **動きの「形」**（重力非依存の関節列）。重力下の力学（ZMP・安定性）は
別途 Phase 1.5「計測再生(Logging)」で立位・torque ON 実走時の IMU と突き合わせて取得する。
詳細は [docs/DESIGN.md](../../docs/DESIGN.md)。
